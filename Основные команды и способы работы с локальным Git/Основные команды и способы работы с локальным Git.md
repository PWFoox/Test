## 1. Инициализация репозитория

```bash
# Создание нового репозитория
git init

# Проверка статуса репозитория
git status
```

## 2. Работа с файлами и индексом

```bash
# Добавление файла в индекс
git add filename.txt

# Добавление всех файлов
git add .
git add --all

# Добавление файлов по маске
git add *.js

# Удаление файла из индекса
git rm --cached filename.txt
```

## 3. Создание коммитов

```bash
# Создание коммита с сообщением
git commit -m "Описание изменений"

# Добавление файлов в индекс и создание коммита
git commit -am "Описание изменений"

# Изменение последнего коммита
git commit --amend
```

## 4. Просмотр истории

```bash
# Просмотр истории коммитов
git log

# Компактный вид истории
git log --oneline

# История с графическим отображением веток
git log --graph --oneline --decorate

# Просмотр изменений в файлах
git diff
git diff filename.txt
```

## 5. Работа с ветками

```bash
# Создание новой ветки
git branch branch-name

# Переключение на ветку
git checkout branch-name

# Создание и переключение одной командой
git checkout -b new-branch

# Просмотр списка веток
git branch

# Удаление ветки
git branch -d branch-name
```

## 6. Слияние веток

```bash
# Слияние указанной ветки в текущую
git merge branch-name

# Отмена слияния в процессе конфликта
git merge --abort
```

## 7. Отмена изменений

```bash
# Отмена изменений в рабочей директории
git checkout -- filename.txt

# Отмена изменений в индексе
git reset HEAD filename.txt

# Отмена последнего коммита с сохранением изменений
git reset --soft HEAD^

# Полная отмена последнего коммита
git reset --hard HEAD^

# Создание нового коммита, отменяющего изменения
git revert commit-hash
```

## 8. Работа с метками (tags)

```bash
# Создание легковесной метки
git tag v1.0

# Создание аннотированной метки
git tag -a v1.0 -m "Version 1.0"

# Просмотр меток
git tag

# Удаление метки
git tag -d v1.0
```

## 9. Сохранение изменений во временное хранилище (stash)

```bash
# Сохранение текущих изменений
git stash

# Просмотр списка сохранений
git stash list

# Применение последнего сохранения
git stash apply

# Применение конкретного сохранения
git stash apply stash@{2}

# Удаление последнего сохранения
git stash drop
```

## 10. Настройка Git

```bash
# Установка имени пользователя
git config --global user.name "Your Name"

# Установка email
git config --global user.email "your.email@example.com"

# Просмотр настроек
git config --list
```

## Полезные советы:

1. Используйте осмысленные сообщения коммитов, описывающие внесенные изменения
2. Регулярно создавайте коммиты для логического разделения изменений
3. Используйте ветки для разработки новых функций
4. Перед слиянием веток убедитесь, что текущая ветка актуальна
5. При работе в команде согласуйте правила именования веток и формат сообщений коммитов

## Справочная таблица основных команд Git

| Команда          | Описание                                   | Основные флаги и их значение                                                                                                                                                                                         |
| ---------------- | ------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `git init`       | Инициализация нового Git репозитория       | `--bare` - создание репозитория без рабочей директории<br>`--quiet` - подавление вывода сообщений                                                                                                                    |
| `git status`     | Просмотр состояния рабочей директории      | `-s, --short` - краткий формат вывода<br>`-b` - показать информацию о ветке<br>`--ignored` - показать игнорируемые файлы                                                                                             |
| `git add <file>` | Добавление файла в индекс                  | `-A, --all` - добавить все файлы<br>`-p` - интерактивное добавление частей файлов<br>`-n` - "сухой запуск" без реального добавления файлов                                                                           |
| `git commit`     | Создание коммита                           | `-m "message"` - добавление сообщения<br>`-a` - автоматическое добавление изменённых файлов<br>`--amend` - изменение последнего коммита<br>`--no-edit` - использовать предыдущее сообщение коммита                   |
| `git log`        | Просмотр истории коммитов                  | `--oneline` - компактный формат<br>`--graph` - показать графическое представление<br>`--decorate` - показать ссылки и теги<br>`-p` - показать изменения в коммитах<br>`-n <number>` - ограничить количество коммитов |
| `git branch`     | Просмотр списка веток                      | `-d` - удаление ветки<br>`-D` - принудительное удаление ветки<br>`-m` - переименование ветки<br>`-a` - показать все ветки (включая удалённые)<br>`-v` - показать последний коммит каждой ветки                       |
| `git checkout`   | Переключение между ветками/коммитами       | `-b` - создать и переключиться на новую ветку<br>`-f` - принудительное переключение<br>`--track` - установка отслеживания удалённой ветки                                                                            |
| `git merge`      | Слияние веток                              | `--no-ff` - создать коммит слияния всегда<br>`--squash` - объединить все коммиты в один<br>`--abort` - прервать слияние при конфликте                                                                                |
| `git pull`       | Получение и слияние изменений              | `--rebase` - применить изменения через rebase<br>`--no-commit` - не создавать автоматический коммит<br>`--squash` - объединить все изменения в один коммит                                                           |
| `git push`       | Отправка изменений в удалённый репозиторий | `-u` - установка отслеживания удалённой ветки<br>`--force` - принудительная отправка<br>`--tags` - отправка тегов                                                                                                    |
| `git stash`      | Временное сохранение изменений             | `save "message"` - сохранить с сообщением<br>`pop` - применить и удалить последнее сохранение<br>`apply` - применить без удаления<br>`drop` - удалить сохранение<br>`list` - показать список сохранений              |
| `git reset`      | Отмена изменений                           | `--soft` - сохранить изменения в индексе<br>`--mixed` - сбросить индекс, сохранить файлы<br>`--hard` - полный сброс изменений                                                                                        |
| `git tag`        | Управление метками                         | `-a` - создать аннотированный тег<br>`-d` - удалить тег<br>`-l` - список тегов<br>`-m` - добавить сообщение к тегу                                                                                                   |
| `git diff`       | Просмотр различий                          | `--staged` - изменения в индексе<br>`--word-diff` - показать различия по словам<br>`--stat` - статистика изменений                                                                                                   |
| `git clone`      | Клонирование репозитория                   | `--depth` - ограничение глубины истории<br>`--branch` - клонировать конкретную ветку<br>`--bare` - клонировать без рабочей директории                                                                                |
